# API Guide

Auron platform provides full support for REST APIs and let users perform all the operations on their behalf using API key bearer token.

All the operations done via REST APIs are performed on behalf of the user based on API key. API keys are generated by user and are always scoped to a user.

# Getting Started

Auron platform offers two different environments which are `dev` for development and testing purposes and a `prod` which is a production environment.

For the `dev` envoironment the base URL for api calls is `https://dev.useauron.ai/api/v1`.
For `prod` envoironment you need to get input for based URL for apis as urls are generated based on tennant id and user need to manually insert the base URL for apis in case for `prod`.

# Authentication

All the API endpoints are protected using JSON WEB Tokens. Users first need to generate a token by logging into their accounts and go inside profiles settings and scroll down to find "Your API Keys" and generate a new API key and paste here.

These JSON Web Tokens are scoped to a user profile and must be sent as `Authorization` header as a bearer token.

Example Snippet

```
Header: {
    Authorization: `Bearer <USER_TOKEN>
}
```

# Rules

As all the operations can be performed using REST APIs it is important for mutation APIs like POST, PUT and DELETE operations to once get an approval from user before executing the call.

Always render a structured markdown text showcasing user and asking approval if user wants to proceeed with same or not.

For any POST (Create) calls, fetch existing records, entries of same thing and analyse the structure and already present resources. If you think user is trying to make duplicate resource, you can warn the user as well for same thing.

Fetching existing resources will help you understand the structure better for the resource and make better POST calls.

# API Refrence and Endpoints

All the available APIs and documentations are present on `https://dev.useauron.ai/api/api-docs` for development environment and for production environment the URL would be `https://{{BASE_URL}}/api/api-docs` where {{BASE_URL}} is the url provided by the user's live instance.

For API Refrence always fetch a fresh copy of OPENAPI.json which has refrence of all the available endpoints and schema definations. OPENAPI.json file is present at `https://dev.useauron.ai/api/v1/openapi.json`for development environment and for production environment the URL would be `https://{{BASE_URL}}/api/v1/openapi.json` where {{BASE_URL}} is the url provided by the user's live instance.

Code snippet:

```bash
 curl -sL https://{{BASE_URL}}/api/v1/openapi.json | jq
```

Note: Always ask user about environment they want to use and based on that make fetch calls.
For example in `dev` env the base url for openapi.json file is `https://dev.useauron.ai/api/v1/openapi.json`

Always run the above command to fetch fresh copy of openapi.json file and use grep to find relevant endpoints and call them as per users need.

# Storage and User Prefrences

To store and read user prefrences such as env, token and API keys use the `storage.js` file.

Useage: `node storage.js {{action_name}} {{data}}`

Supported action_name:

- `getConfiguration`: To read configuration
- `setConfiguration`: To set update key configuration

For Example:

To read user configuration:

```bash
node storage.js getConfiguration
```

Output:

```json
{
    "env:" dev, // Selected environment for API calls,
    "authentication": {
        "apiKey": "xxxx" // User's API Key
    },
    "state": { // State Object for key value pair storage
        "organizationId": "xxx", // user's selected organization id,
    }
}
```

To set user configuration:

```bash
node storage.js setConfiguration "{ "env": "dev" }"
```
